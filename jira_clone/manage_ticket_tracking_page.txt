Front End Flow : 
	- System will create a status card column for each  status.
	- System will create a Ticket card for  each sprint ticket.
	- System will put each  Ticket card in the correct status card column.
End Front End Flow.

	capability :

End Display.
	
User Actions : 

	01 - The System should allow user to change ticket current status. 

	Actions - 01 : 
		Name : changeTicketState 
		Pre Condition: 
		Main : 
			00.01- User should choose the target ticket so System(frontEnd) highlight the element that he  can drag to (the allowed target is  just frontEnd calculated without backEnd call ). 
			00.02- User drop the ticket in a target state.
		anomaly flow : 
			01.01 - user drop the ticket in the same current state so system will remove the highlight effect.
		
		Post Condition: 
			success : the ticket will appear in the new current state.
			failed : ticket will stay  in the same  current state with show a message error.
		

	00.02 - request : 
		Name : changeTicketState 
		Input : ticketId, toStatusId, wokflowId, workflowTransitionId  
		
		
	Sequence : 
		- controller will receive the request. 
		- controller will validate the input ticketId , toStatusId.
		- controller will call the WorkflowService . changeTicketState
		
		* - Workflow Service calling the DomainCompleteValidator check if the Ticket Type exist. 
			failed : throw a exception 
		* - Workflow Service calling the DomainCompleteValidator check if this Workflow is assigned to this Ticket Type.
			failed : throw a exception 
		* - Worlflow Service calling the DomainCompleteValidator check if this workflow transition owned by this workflow. 
			failed : throw a exception 
		* - Workflow Service will validate for  <canTransition> canTransition(fromStatusId , toStatusId , workflow transition <pass the full object as we get from the previous step>)  is a method in workflowService. 
			failed : throw a exception 
		- Workflow Service will update the ticket current status to the new one.
		
		- controller will encapsulate the  error or success flow in ApiResponse.
