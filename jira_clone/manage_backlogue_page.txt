	What is  sprint ? 
			A project can have {0,many} sprint.
	
			The Sprint have : 
				- start time 
				- end time
				- name 
				- goal (description)
				- record_status : (future, complete , delete , in_progress)
				- PROJECT_ID RELATIONSHIP
			The sprint is complete when: 
				- user manually click complete 
				- achieve end time 
			
			
		What is ticket container ? 
			Just a display way for the ticket.
			a ticket container show a button to add an epic for this ticket if it was not added else it show the parent epic.
			a ticket container show a arrow button that show all sub task container when clicked.
			a ticket container show the title , summary , ticket type Name , current state name , story point , priority, the assigned member for this ticket.
			a ticket container show the  epic name pather only if it has  a epic pather else it show a button to add a epic.
			a ticket container show a combo box so it was selected.
			a ticket container show a edit button right to summary to  update it. 
			>> so it was not an entity.
		

		What is a sub task container ? 
			Just a display way for the sub task.
			a sub task container show the title , summary , current state name , story point , priority, the assigned member for this ticket.
			a sub task container show a combo box so it was selected.
			a sub task container show a edit button right to summary to  update it. 
			>> so it was not an entity.

			- Display a create button to create a new sprint. 
			- Display a create button to create a new ticket.
			- Display a comboBox to select all ticket container.
